15 Detection Rules

Every rule uses tree-sitter AST queries — not fragile regex patterns.

🐍 Python

PY001 unused-import — Imported module never used in the file
PY002 hallucinated-func — Function called but not defined or imported
PY003 deprecated-api — Uses removed APIs like pkg_resources, distutils
PY004 wrong-arg-order — Suspicious or duplicate keyword arguments
PY005 type-mismatch — str + int, len() compared to string

🌐 JavaScript / TypeScript

JS001 unused-import — Imported symbol never referenced
JS002 hallucinated-func — Function call with no definition
JS003 deprecated-api — substr(), createClass, console.exception
JS004 wrong-arg-order — Callback (data, error) vs (error, data)
JS005 null-undefined — Property access without null check

🔧 Generic (All Languages)

GEN001 hardcoded-creds — API keys, passwords, tokens, DB URLs
GEN002 infinite-loop-risk — while True without break/return
GEN003 unused-variable — Variable assigned but never read
GEN004 commented-deadcode — Large blocks of commented-out code
GEN005 magic-number — Same literal 3+ times, no named constant

How It Works

Tree-sitter parses your code into an AST. Rules inspect the AST for AI hallucination patterns.

Parse

aiverify reads your file and builds a full Abstract Syntax Tree using tree-sitter — the same parser used by IDEs like VS Code.

Analyze

15 rules run against the AST, each looking for specific patterns that indicate AI-generated code: invented functions, deprecated APIs, type mismatches, and more.

Report

Results are displayed with color-coded severity. JSON output available for CI/CD. Exit code 1 blocks bad code from shipping.

Usage

Designed for both local development and CI/CD pipelines.

aiverify --help
# Scan a file or directory
aiverify app.py
aiverify src/

# JSON output for CI/CD
aiverify src/ --json

# Run specific rules only
aiverify app.py --rules PY002 GEN001

# List all 15 rules
aiverify --list-rules

# Generate config file
aiverify --init
